# 🛠️ FiveMスクリプトの基本的な導入手順とserver.cfgの書き方（ESX / QBCore）

Keymasterからスクリプトをダウンロードしたら、次はご自身のFiveMサーバーへの導入（インストール）です。
このページでは、当ストアのスクリプトを含む、一般的なFiveMスクリプトの基本的な導入手順をステップバイステップで解説します。

## ステップ 1：リソース（resources）フォルダへの配置

1. Keymasterからダウンロードした `.zip` ファイルを解凍します。
2. 解凍したフォルダの中に `fxmanifest.lua` というファイルが入っていることを確認してください。
   ※フォルダを開いてさらに同じ名前のフォルダがある場合は、一番内側（`fxmanifest.lua` が直接入っている階層）のフォルダを使用します。
3. ご自身のFiveMサーバーの `resources` フォルダを開き、解凍したフォルダを配置します。
   **💡 おすすめの管理方法:** `[standalone]` や `[syndicate]` のように `[` `]` で囲んだカテゴリ用のフォルダを作り、その中に入れると後から管理しやすくなります。

## ステップ 2：データベース（SQL）のインポート
※スクリプト内に `.sql` ファイル（例：`install.sql`）が同梱されていない場合は、このステップを飛ばしてステップ3へ進んでください。

アイテムやギャングのデータなどを保存するため、データベースに専用のテーブルを作成します。

1. **HeidiSQL** や **phpMyAdmin** などのデータベース管理ソフトを開きます。
2. ご自身のサーバーのデータベース（ESXやQBCoreのデータが入っている場所）を選択します。
3. 同梱されている `.sql` ファイルを読み込み、実行（クエリを実行）します。
4. エラーが出ず、新しいテーブルやアイテムが追加されていれば成功です。

## ステップ 3：server.cfg への記述（最重要）

スクリプトをサーバー起動時に読み込ませるため、`server.cfg` を編集します。
ここで最も重要なのが **「記述する順番（起動順）」** です。前提となるスクリプト（Dependencies）を先に読み込ませないとエラーになります。

1. サーバーのルートディレクトリにある `server.cfg` をテキストエディタで開きます。
2. `ensure`（または `start`）コマンドを使って、スクリプト名を追記します。

**▼ server.cfg の記述例**
```cfg
# --- 必須の前提スクリプト（必ず先に読み込む） ---
ensure ox_lib
ensure ox_target
ensure ox_inventory

# --- フレームワーク ---
ensure es_extended  # (QBCoreの場合は ensure qb-core)

# --- 当ストアのスクリプト ---
ensure [ここに配置したフォルダ名を正確に入力]
